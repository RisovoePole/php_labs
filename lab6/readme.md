## Цель работы

Освоить работу с HTML-формами в PHP: создание формы, отправку данных методом `POST`, серверную обработку, базовую валидацию, хранение данных в читаемом формате JSONL и вывод записей в виде HTML-таблицы с сортировкой.

## Постановка задачи

Необходимо разработать простую систему добавления и просмотра рецептов. Пользователь заполняет форму, данные отправляются на сервер, проверяются валидатором и сохраняются в файл. После этого записи можно просматривать в таблице и сортировать по нескольким полям.

## Краткое описание решения

В работе реализованы следующие части:

- HTML-форма создания новой записи в [lab6/index.php](lab6/index.php).
- Серверная обработка данных в [lab6/logic.php](lab6/logic.php).
- Интерфейс валидатора в [lab6/ValidatorInterface.php](lab6/ValidatorInterface.php).
- Класс валидатора в [lab6/Validator.php](lab6/Validator.php).
- Вывод всех сохранённых записей в [lab6/list.php](lab6/list.php).

Данные сохраняются в формате JSONL в файл `data/recipes.jsonl`, где каждая строка - отдельная запись. Это удобно для последующего чтения построчно и для простого расширения проекта.

## Ход выполнения работы

### Шаг 1. Определение модели данных

В качестве модели данных был выбран рецепт. Для одной записи используются следующие поля:

- `title` - название рецепта;
- `category` - категория рецепта;
- `date` - дата приготовления или публикации;
- `ingredients` - список ингредиентов;
- `instructions` - инструкция по приготовлению;
- `created_at` - время создания записи на сервере.

Такой набор полей позволяет показать и ввод текста, и выбор из списка, и работу с датой.

### Шаг 2. Создание HTML-формы

Форма размещена в [lab6/index.php](lab6/index.php). Она отправляет данные методом `POST` в [lab6/logic.php](lab6/logic.php).

Форма содержит:

- поле `title` типа `text`;
- поле `category` типа `select`;
- поле `date` типа `date`;
- поле `ingredients` типа `textarea`;
- поле `instructions` типа `textarea`.

Для улучшения пользовательского опыта добавлена базовая клиентская валидация через HTML-атрибуты:

- `required`;
- `minlength`;
- `maxlength`.

Пример формы:

```php
<form action="logic.php" method="post" novalidate>
 <label for="title">Название (обязательно)</label>
 <input type="text" id="title" name="title" required minlength="2" maxlength="100">

 <label for="category">Категория</label>
 <select id="category" name="category">
  <option value="">-- выберите --</option>
  <option value="breakfast">Завтрак</option>
  <option value="lunch">Обед</option>
  <option value="dinner">Ужин</option>
  <option value="dessert">Десерт</option>
 </select>

 <label for="date">Дата (YYYY-MM-DD)</label>
 <input type="date" id="date" name="date" required>
 ...
</form>
```

### Шаг 3. Обработка данных на сервере

Файл [lab6/logic.php](lab6/logic.php) принимает данные из `$_POST`, формирует массив записи и передаёт его в валидатор.

Если найдены ошибки, пользователь возвращается на страницу формы с сообщением об ошибке. Если ошибок нет, запись сериализуется в JSON и добавляется в файл `data/recipes.jsonl`.

Сохранение выполнено в формате JSONL, потому что:

- одна строка соответствует одной записи;
- файл удобно читать построчно;
- формат легко расширять и обрабатывать без сложного парсинга;
- данные остаются читаемыми для человека.

Фрагмент логики сохранения:

```php
$line = json_encode($data, JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES);
file_put_contents($file, $line . PHP_EOL, FILE_APPEND | LOCK_EX);
```

### Шаг 4. Вывод данных

Для просмотра сохранённых записей создан файл [lab6/list.php](lab6/list.php).

Этот скрипт:

- читает `data/recipes.jsonl` построчно;
- преобразует каждую строку из JSON в массив;
- выводит все записи в HTML-таблице;
- поддерживает сортировку по полям `title`, `category`, `date`, `created_at`.

Для полей с датами сортировка выполняется через `strtotime()`, чтобы порядок был корректным по времени, а не только по строковому сравнению.

### Модификация системы валидации

Чтобы архитектура была расширяемой, была введена абстракция в виде интерфейса [ValidatorInterface.php](lab6/ValidatorInterface.php).

В интерфейсе и в классе [Validator.php](lab6/Validator.php) каждая проверка вынесена в отдельный метод:

- `validateTitle()`;
- `validateCategory()`;
- `validateDate()`;
- `validateIngredients()`;
- `validateInstructions()`;
- `validate()`.

Метод `validate()` только агрегирует результаты отдельных проверок. Такой подход упрощает поддержку кода и позволяет позже добавить новые валидаторы без переписывания основной логики.

## Контрольные вопросы

### 1. Какие существуют методы отправки данных из формы на сервер? Какие методы поддерживает HTML-форма?

Основные методы отправки данных из HTML-формы на сервер - это `GET` и `POST`.

- `GET` передаёт данные в URL, в строке запроса. Этот метод подходит для поиска, фильтрации и действий, где важно видеть параметры в адресной строке.
- `POST` передаёт данные в теле запроса. Этот метод используют для отправки форм с большим количеством полей, а также для действий, которые изменяют данные на сервере.

HTML-форма поддерживает в первую очередь именно эти два метода через атрибут `method` тега `<form>`.

### 2. Какие глобальные переменные используются для доступа к данным формы в PHP?

Для доступа к данным формы в PHP используются суперглобальные массивы:

- `$_GET` - данные, отправленные методом `GET`;
- `$_POST` - данные, отправленные методом `POST`;
- `$_REQUEST` - объединённый доступ к данным из `GET`, `POST` и `COOKIE`, но использовать его нужно осторожно, чтобы не смешивать разные источники без необходимости.

В данной лабораторной работе данные формы читаются из `$_POST`, потому что форма отправляется методом `POST`.

### 3. Как обеспечить безопасность при обработке данных из формы (например, защититься от XSS)?

Безопасная обработка данных формы строится на нескольких правилах:

- проверять данные на сервере, а не только через HTML-атрибуты;
- ограничивать длину и формат полей;
- использовать белые списки допустимых значений для полей с фиксированными вариантами;
- экранировать вывод пользовательского текста через `htmlspecialchars()`;
- не вставлять пользовательский ввод в HTML, JavaScript или SQL без обработки.

Для защиты от XSS перед выводом данных на страницу следует использовать, например, такой код:

```php
echo htmlspecialchars($value, ENT_QUOTES | ENT_SUBSTITUTE, 'UTF-8');
```

Это предотвращает выполнение опасного HTML и JavaScript-кода в браузере пользователя.
